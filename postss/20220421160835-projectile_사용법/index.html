projectile 무엇?

  • emacs에서 program을 작성한다는 것은 project를 만든다고 생각하면 된다. 하나의 program은 하나의 project로 봐도 무방하며, 하나의 git repository로 봐도 무방하다.
  • git을 clone해서 emacs에서 보면 하나의 projectile로 자동으로 관리된다. 만일 별도의 폴더를 만들고 projectile로 관리할 수도 있지만 권장되진 않는다. 따라서 무조건 git repository를 만들고 이를 local에 down받은후 emacs에서 열게 되면, projectile로 관리된다.
  • git에서 repo를 만들고, 이를 다운 받아서 emacs에서 열면 projectile로 관리된다는 것을 알았다. program짤 일이 있으면, 이렇게 git repo부터 만들자. 그런데, 이렇게 projectile로 관리된다는게 뭔 소용이 있는지 부터 알아야 할 것 같다. 좋은점이 있으니까 할테니 말이다.

projectile external tools

3개의 tools을 terminal에서 설치해야 한다. 이것을 설치해야 하는이유는 grep을 위해서다. projectile에서 가장 많이 사용하는 기능 중의 하나가, projectile에서 특정 단어를 찾을 수 있는 grep이기 때문이다.

  • brew install fd ag, rg for projectile-ag and projectile-ripgrep.

projectile 사용법

  • (C-c p f: find file) project에 있는 모든 file들을 보여준다. 일일이 폴더를 뒤져가면서 찾을 필요가 없다. project에서 사용되는 모든 file을 한번에 보여준다. 즉 file이름만 알면 subdirectory를 이동해가면서 찾을 필요가 없다. 편리하다.
  • (C-c p p: find projectile)이다. projectile로 이동하게 해준다. projectile 목록이 보이고 선택하면 된다.
