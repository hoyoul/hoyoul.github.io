big O 표기법

알고리즘을 수행할때 걸리는 시간을 “오래걸린다”, “금방 끝난다"로 나타낼 수 없다. 수행하는 함수에 대한 시간 복잡도는 보통 for-loop의 개수로 나타낸다. 그리고 최고차항만을 고려한다.

for i in range(10):
  print(i)

10개의 data모두에 대해 10번의 연산이 일어난다. n개의 data에 대해서 n번의 연산이 수행. O(n)으로 나타낼수 있다.

for i in range(10):
  print(i)
  print(i)

10개의 data에 2x10의 연산이 일어난다. n개의 data에 대해서 2n번의 연산이 수행된다. O(n)으로 나타낸다.

string pool

이장에서 결국 얘기하고 싶은것은 string pool이 아닌가 한다. string pool은 string literal의 중복이 메모리의 낭비를 가져오기 때문에 대부분의 언어에서 지원을 한다. lisp의 경우 interning을 통해서 string literal을 string pool에 넣을 수 있고, *obarray*를 통해서 string pool에 있는 모든 문자열을 볼수 있다. 반면에 python은 지원하지 않는다.

mutable과 immutable

python책에서 mutable과 immutable을 얘기하는데 솔직히 이해를 못하겠다. python은 functional language와는 달리 모든게 object이다. 따라서 symbol이 가질수 있는 값은 주소값이다. functional language에서는 value를 갖는다. 따라서 assignment의 동작방식이 다르다. lisp이나 haskell같은 언어는 symbol이 값을 갖는다. evaluation을 하면 값이 나오지만, oop언어는 다르다. assignment에서 evaluation을 하는 경우 객체의 주소가 나온다. 예를 들어보자. 아래는 mutable과 immutable을 설명하는 standard한 예제다. 그런데 assignment의 개념과 evalutation의 개념만 알면 mutable이나 immutable의 개념이 필요한지 모르겠다. 용어의 남발은 삶을 피곤하게 한다.

# immutable 객체의 예제
a = 5
b = a
a = a + 2

print(a)  # 출력 결과: 7
print(b)  # 출력 결과: 5


# mutable 객체의 예제
c = [1, 2, 3]
d = c
c[0] = 5

print(c)  # 출력 결과: [5, 2, 3]
print(d)  # 출력 결과: [5, 2, 3]
