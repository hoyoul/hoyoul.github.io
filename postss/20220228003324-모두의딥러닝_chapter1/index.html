(2) 미지의 일을 예측하는 원리

  • 기존 환자의 데이터를 이용해 새로운 환자의 생사를 예측하는 프로그램을 짜봐!
  • [내 생각] 내가 아무런 배경지식이 없는 상태에서 이 문제를 스스로 해결할 수 있을까? 누군가가 내게 이런 질문을 했을때, 어떤 데이터도 주어지지 않고, 문제만 주어졌을때 나는 어떤 생각으로 이 문제를 풀 것인가? 바보같지만,이 문제를 나는 토트넘과 첼시의 경기에서 토트넘이 이길 확률을 알수 있는가?로 바꿔 생각해보기로 한다. 내가 필요한 데이터는 토트넘과 첼시의 승패가 담긴 모든 데이터다. 그런 데이터를 가져와서 통계를 내서 사용하는 것이다. 그래서 토트넘이 첼시를 이길 확률은 60%라고 예측할 것이다. 너무 바보같지만, 그 생각 밖에 나지 않는다. 내가 아는 미래를 예측하는 또 다른 방법은 Markov property가 있는 경우다. 이 경우 매번 예측은 현재상태와 어떤 연관이 있어야 한다. 토트넘이 직전 경기에서 이겼다고 해서 다음 경기에서 이긴다는 보장이 있나? 어떤 관계가 있나? 그런데 여기에선 아무런 관계가 없다. 따라서 이 방법도 쓸 수 없다.
  • [내생각2] 이제 이 책에서 제시하는 방법과 내 머리속의 바보같은 방법들의 차이를 살펴보자. 우선 data에 대한 생각이 틀려먹었다. 환자의 data는 수많은 항목으로 이루어졌다. 혈액형, 심박수, 기침,흡연, 천식 유무등 많은 항목의 data를 가지고 있다는 것이다. 즉 우리가 토트넘과 첼시의 경기를 예측한다고 할때, 과거의 승패 data만 사용하겠다는 생각은 너무나도 naive하다. 데이터가 종류별로 많아야 한다. 예를 들어, 잔디상태, 선수들의 피지컬, 전술전략…될 수 있는대로 모든 항목을 데이터로 할 수 있어야 한다. 두 번째로, 토트넘과 첼시라는 팀자체가 중요하지 않을 수도 있다. 새로운 환자의 생사를 예측할 때, 우리가 사용한 데이터는 기존환자의 정보였다. 새로운 환자와 기존환자의 공통점은 환자라는 것이고, 환자에서 얻을 수 있는 특성들을 데이터로 사용했다. 그렇기 때문에 새로운 환자에 대한 예측을 할 수 있었던 것이다. 토트넘과 첼시도 비슷하게 생각해야 한다. 즉 data는 축구경기가 갖는 특성들을 데이터로 사용해야 한다. 축구경기가 공통적으로 갖는 feature(attribute)를 사용한 데이터를 사용해야만, 우리는 맨유와 첼시, 울버햄튼과 토트넘의 경기를 예측할 수 있다. 마지막으로 환자의 feature를 가진 데이터의 결과에 해당하는 데이터가 필요하다. 토트넘 경기예측에서 나는 바보 같이, 축구 결과 data만 사용하려고 했는데, 이것이 완전히 잘못된 data는 아니다. 왜냐면 이게 결과 data에 해당하기 때문이다. 이 결과 데이터를 label이라고 부르기도 한다.
  • [내생각3] 이제 data에 대한 나의 바보같은 생각은 바로잡았다. 새로운 환자의 생사여부를 예측하기 위해선, 환자가 가진 수많은 항목들로 이루어진 data와 환자가 죽었다/ 살았다라는 data가 필요하다는 것을… 이제 data가 준비되었다. 그러면 여기서 어떻게 예측할 수 있는가? 나는 언뜻 떠오르지 않는다. 환자 data중에 결과 data에 영향을 미치는 항목으로 사망/생존을 판단하면 되지 않을까? 하는 생각은 하지만, 그 수많은 데이터 항목이 있고, 수많은 레코드가 있을때, 어떻게 그 인과 관계를 찾는 다는 말인가? 책에서는 다른 방식을 사용한다. 환자 data가 100개 항목이 있다면, 100개의 axis를 갖는 공간을 만들고, 거기에 환자 data를 한 점으로 표시한다. 이렇게 한후 각각의 점에 label을 줄수 있다. 사망은 빨간점, 생존은 파란점. 이렇게 해서 사망과 생존을 나누는 어떤 선, 혹은 평면을 찾는다. 이렇게 기준이되는 평면 or 선을 만들면, 우리는 새로운 환자에 대해서 사망/생존을 판단할 수 있다는 것이다. feature(attribute0에 따른 공간을 만드는것은 엄청난 계산을 필요로 한다. 그런데 이런 계산은 computer가 알아서 해준다는 것이다. 이것이 인공지능(머신러닝)을 이용하는 방법이다. 이제 책을 요약하도록 하자.

(4) 폐암 수술 환자의 생존율 예측하기

from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense

import numpy as np
import tensorflow as tf

np.random.seed(3)
tf.random.set_seed(3)

Data_set = np.loadtxt("./deeplearning/dataset/ThoraricSurgery.csv", delimiter=",")

X = Data_set[:,0:17]
Y = Data_set[:,17]

model = Sequential()
model.add(Dense(30, input_dim=17, activation ='relu'))
model.add(Dense(1, activation = 'sigmoid'))

model.compile(loss= 'binary_crossentropy', optimizer='adam', metrics=['accuracy'])
model.fit(X,Y, epochs=100, batch_size=10)

numpy package (import numpy as np)

numpy package란? numpy package가 없으면 안되는가?

  • python에서 취급하는 data는 대부분 1차원이다. file, memory가 sequence한 1차원 배열이기 때문이다. 기껏해야 graphics에서 rgb파일 처리 때문에 list,tuple,dictionary로 3차원의 데이터를 표현하고 사용할 뿐이다. 그런데, deep learning에서 사용하는 feature(attribute)데이터들은 다차원의 공간을 사용하고 연산을 한다. 쉽게 말해서 linear algebra 연산을 많이 사용하고 표현한다.따라서 numpy library의 도움이 필요하다.사족이긴 하지만, 기존의 python list로 못하는건 아닌데, 복잡하다. 예를들어서 list로 1차원,2차원,3차원 데이터를 만들어 보면 다음과 같다.
import numpy as np

a = [1,2,3,4]
print(a)
b = [[1,2],[2,3],[3,4]]
print(b)
c = [ [[1,2],[2,3],[3,4]], [[1,2],[5,6],[7,8]]]
print(c)
  • numpy의 reshape을 사용해보자. 다차원의 데이터를 쉽게 만들어 사용할 수 있음을 볼 수 있다.

numpy random seed

  • 난수를 예측 가능하게 해준다. seed가 같다면, 동일한 난수를 발생한다.

  • seed값은 의미가 없다.

    import numpy as np
np.random.seed(0);np.random.rand(4)

    np.random.seed(0);np.random.rand(4)

tensorflow random set_seed

  • seed가 같으면 동일한 난수 발생. numpy에서 random의 seed를 설정하는 것과 같다.

code에서 random을 사용하는 부분은 어디에 있는가? 그리고 왜 사용하는가?

  • model에서 tensorflow의 random()를 사용해서 parameter를 설정한다고 한다. seed를 고정해서 동일한 parameter를 사용하는 이유는 재현성(reproducibility)과 관련있다. 매번 실행할때 마다, 결과가 다르게 나오면 신뢰성이 떨어지기 때문이다.
  • 예를 들어보자.
import tensorflow as tf

tf.random.set_seed(2)
model = tf.keras.models.Sequential([
tf.keras.layers.Input(shape=(2,)),
tf.keras.layers.Dense(4, activation='tanh'),
tf.keras.layers.Dense(1, activation='sigmoid')
])
print('모델의 parameters', model.get_weights())

tf.random.set_seed(2)
model = tf.keras.models.Sequential([
tf.keras.layers.Input(shape=(2,)),
tf.keras.layers.Dense(4, activation='tanh'),
tf.keras.layers.Dense(1, activation='sigmoid')
])
print('모델의 parameters', model.get_weights())
  • tensorflow random seed를 설정해서 매번 테스트시에 같은 결과를 가져오는 것을 보았다. 하지만, numpy의 rand함수가 모델에 사용되는거 같지는 않다. numpy의 seed를 설정하는게 의미가 없어보인다.

numpy loadtxt (Thoraricsurgery)

  • Thoraricsurgery라는 파일을 보면, 470라인으로 이루어진 csv 파일이다.

  • loadtxt로 읽으면, ndarray로 바꿔진다. 즉 470x18의 행렬을 return하게 된다. 아래 출력을 확인해 보자.
import numpy as np
Data_set = np.loadtxt("./deeplearning/dataset/ThoraricSurgery.csv", delimiter=",")
print(Data_set.shape)
print(Data_set)

Data_set[:, 0:17], Data_set[:,17] 의 의미

  • ndarray slicing: list의 slicing과는 좀 다르다. ndarray는 다차원 배열을 나타내기 때문에, slicing이 다차원 배열에 맞게 slicing할 수 있다. Data_set이 470x18의 2차원 배열이기 때문에 첫번째 slicing인수는 row를 뜻하고, 두번째 인수는 column을 의미한다. 따라서, 470 rows에서 17개의 column을 가져오겠다는 뜻이다.
  • Data_set의 구성은 다음과 같이, 0-17개의 column은 환자의 data를 나타내고, 18번째 column은 label(class)을 의미한다.
  • fit할 때 보면, deep learning의 입력으로 feature를 주고, 출력값으로 class를 준다.

Sequential이란?

  • sequential은 model을 만든다. Sequential model은 model.add()를 통해서 layer를 쌓을수 있게 해준다.

Dense()란?

  • model.add()로 layer를 만들때, layer의 특성을 Dense라는 함수에 기술한다.
