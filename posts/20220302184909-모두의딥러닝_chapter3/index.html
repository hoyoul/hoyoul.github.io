1 Linear Regression (선형회귀)

1.1 (1) 선형 회귀의 정의

  • Linear Regresssion(책에서 정의): 가장 훌륭한 예측선 긋기
  • 학생들의 중간고사 성적이 다 다르다. -> 학생들의 중간고사 성적이 [ ] 에 따라 다르다.
  • 윗 문장에서 독립변수에 의해 종속변수가 결정되는 관계를 볼 수 있다.
  • 안에 공부시간, 시험당일 컨디션, 사교육비 지출액.
  • Simple Linear Regression: 하나의 독립변수로 종속변수가 설명될때.
  • Multiple Linear Regression: 여러개의 독립변수로 종속변수가 설명.

1.2 (2) 가장 훌륭한 예측선이란?

  • Simple Linear Regression의 예: 학생들의 중간고사 성적이 [공부 시간] 에 따라 다르다.
  • [내생각] chapter1에서 궁금한 것이 있었다. 환자의 상태인 17개의 feature 결과값(class)에 맞춰서 deep learning을 구성했다. 그 때, 이상하게 생각했던것은 17개 각각이 어떻게 선정되었는가? 어떤 기준이 있는가? 환자의 결과와 환자의 상태를 나타내는 17개 feature 각각이 결과값과 어떤 의존관계가 밝혀졌기 때문에 feature로 한것인지, 아니면 의존 관계와 관계없이 feature로 사용한건지 궁금했었다. 위에서 말한 독립변수와 종속변수의 관계가 있었기 때문에 feature로 사용된 것인가? 아니면, 그 feature라는 게 환자의 결과에 영향이 있을 거 같다는 막연한 믿음이나 확신으로 feature로 정했는지 궁금했다. Simple Linear Regression에선 예제로 학생들의 중간고사 성적이 공부시간에 따라 다르다는 것으로 정는데, 이유는 공부시간과 중간고사 성적 사이에 독립변수와 종속변수의 관계가 있기 때문이다. 독립변수와 종속변수의 관계가 있다는것은, 그 식이 1차방정식(linear equation)같은 식으로 표현되어서 x에 어떤값을 넣으면 y값이 나오는 직선과 같은 관계를 갖는다는 것이다. 따라서, 공부시간과 성적이 linear한 관계를 갖고 있다면, 공부시간을 넣으면 어느정도의 성적이 나올것인지 예측할 수 있다는 것이다. 따라서, 우리는 예측선을 찾아야 한다. 예측선중 가장 훌륭한 예측선을 찾아야 한다.

  • 중간고사 성적과 공부시간의 data

  • 중간고사 성적과 공부시간의 graph

  • Goal: 어떤 학생의 공부시간으로 성적을 예측할 수 있겠는가? 우리는 공부시간과 성적이 독립변수와 종속변수의 관계가 linear하다고 생각한다. 따라서 Linear regression방법을 사용한다.

  • [내생각]: chapter1에서 폐암환자의 생존여부를 예측하는 문제와 지금 chapter3에서 공부시간과 성적을 예측하는 문제는 다르다. chapter1에서는 17개의 feature를 deep neural network의 입력으로 넣고, 생존여부가 결과값으로 나왔다. training시에 어떤 polynomial curve를 만들어서 분류하는지, 혹은 linear equation, quadratic equation인지 모른다. 그냥 블랙 박스였다. 그런데 여기서는 데이터를 통해서 직선의 방정식을 만드는 방법을 보여준다.

  • Simple Linear regression에서 찾는 직선

  • 이 직선을 찾는다는 것은 기울기와 절편을 찾는다는 얘기다.

1.3 (3) 최소 제곱법 (the method of least square) 의 수식 유도

  • [내생각] 직선의 방정식에서 2개의 data만 있어도, 두개의 점을 잇는 직선의 기울기와 절편을 찾을수 있다. 3개의 data가 있다면? 직선을 찾을 수 있을까? 찾을 수 없다. 이것은 독립변수와 종속변수에 사용되는 data들이 법칙에 맞게 나오지 않기 때문이다. 즉, data를 그래프로 그렸을 때, data는 직선의 형태가 나오지 않는다. 따라서, 데이터로 직선을 만들수도 없다. 근사화한 직선을 만들어서, 모든 data가 오차는 있지만, 그럭저럭 만족하는 직선을 구해야 한다.어떻게 만들것인가? 오차를 정의해서 이를 이용해 직선을 구한다.즉, 직선과 데이터의 차를 제곱한 오차함수를 만드는데, 이 오차함수는 기울기와 절편에 관한 2차함수로 데이터와 직선간의 거리차가 최소화되는 직선을 말한다. 이것을 편미분해서 기울기와 절편을 구하는데, 이 방식이 최소 제곱법이다. 그런데 책에는 최소제곱법의 공식유도과정은 나와 있지 않다.

  • [참고] 이 책에선 공식만 있을 뿐 공식 유도과정은 빠져있기 때문에 유도 과정을 살펴보자.

  • [요약]

    (1) 최적의 직선을 찾기위해서 직선과 데이터의 오차함수를 구한다.

    (2) 오차함수는 기울기와 절편이 포함된 2차함수로 나타낼 수 있다.

    (3) 2차함수의 최소값에서의 기울기와 절편을 구한다.

    (4) 구해진 직선의 방정식은 오차가 가장 적은 최적의 직선이다.


1.4 (4) 최소 제곱법 (the method of least square) 사용

  • feature가 1개인 경우 최소제곱법 사용, feature가 여러개일 경우 경사하강법을 사용.

  • 최소제곱법의 공식은 다음과 같다.

  • 기울기

  • y절편

  • y절편값

  • 직선의 방정식

  • 직선의 방정식을 사용해서 예측

  • 직선과 data의 그래프

1.5 (5) 코딩으로 확인하는 최소 제곱

  • x: 공부시간, y:점수
import numpy as np
x = [2,4,6,8]
y = [81,93,91,97]

  • 기울기를 구해보자.

  • 분모와 분자에 들어가는 x평균과 y평균부터 구한다.

    mx = np.mean(x)
my = np.mean(y)

  • 분모를 구한다.

    divisor = sum([(i-mx)**2 for i in x])

  • 분자를 구한다.

    def top(x,mx,y,my):
    d = 0
    for i in range(len(x)):
	d +=(x[i] -mx) * (y[i] -my)
    return d
dividend = top(x,mx,y,my)

print("분모:", divisor)
print("분자:", dividend)

  • 기울기를 구한다.

    a = dividend / divisor

  • 절편을 구한다.

    b = my - (mx * a)

  • 출력

    print("기울기 a =",a)
print("절편 b =",b)

1.6 (6) 평균 제곱 오차

  • 최소 제곱 오차는 여러개의 입력을 처리할 수 없다.
  • 여러개의 입력을 처리할 때는, 임의의 선을 그리고 update하는 방식을 사용한다.
  • 오차 평가 알고리즘으로 평균 제곱 오차(Mean Squared Error)가 사용된다.

1.7 (7) 잘못 그은 선 바로잡기

  • “일단 그리고 조금씩 수정해 나가기 방식”
  • 일반적 원리: 가설을 하나 세우고, 이값이 주어진 요건을 충족하는지 판단하여 조금씩 변화를 주고, 이 변화가 긍정적이면, 오차가 최소가 될 때 까지 이과정을 반복.
  • 일단 선을 그린다. 그려진 선에 조금씩 변화를 주어 다른 선을 그린다. 나중에 그린선이 먼저 그린 선보다 더 좋은지 나쁜지를 판단한다. 각 선의 오차를 계산할 수 있어야 하고, 오차가 작은쪽으로 바꾸는 알고리즘이 필요하다.
  • 평균 제곱 오차를 사용해 보자.

[0] Data를 그려보자.

[1] 일단 선을 그린다.

  • 기울기:3, 절편:76으로 y=3x +76의 선을 그린다.

[2] 오차를 확인한다.

  • 선과 data사이의 오차를 확인한다.

  • [참고] 기울기가 크게 잡았을 때 오차

  • [참고] 기울기를 작게 잡았을 때 오차

  • 오차 값의 계산: 예측값(직선값) - 실제 값

[3] 오차 제곱의 합

  • 오차의 합을 구할 때, 오차를 그대로 더하면 안된다. 오차에는 부호가 있다. +1만큼의 오차와 -1만큼의 오차를 더하면 오차가 없는것으로 나온다.

  • 그래서 오차의 제곱을 합한다.

[4] MSE(Mean Sqaured Error)를 구한다.

  • [3]에서 구한 오차 제곱의 합에서, 평균을 구한다.

  • [내생각] 최소제곱법처럼 오차제곱의 합을 그대로 사용해도 될텐데, 평균을 내서 사용하는 이유는 모르겠다. 책에도 설명은 되어 있지 않았다. 인터넷에서 찾아봤을때는, 제곱의 값을 다 더하게 되면 너무 큰값이 나오기 때문에 1/n을 해준거라고 한다. 그렇다면, 기본 원리는 최소제곱법이나 오차제곱이나 동일하다고 생각된다.

1.8 (8) 코딩으로 확인하는 평균 제곱 오차

  • 직선의 parameter를 list로 설정한다. 매번 update를 할 것이다. 
    import numpy as np
fake_a_b = [3,76]
  • data를 만든다. python grammar에서 흥미로운 list comprehension이 사용되었다.
  • [ ( 변수를 활용한 값 ) for ( 사용할 변수 이름 ) in ( 순회할 수 있는 값 )] 
    data = [[2,81],[4,93],[6,91],[8,97]]
x= [ i[0] for i in data ]
y= [ i[1] for i in data ]
  • predict란 이름의 직선의 방정식을 만든다. 
    def predict(x):
    return fake_a_b[0]*x + fake_a_b[1]
  • 평균제곱오차1: list를 입력으로 받아서 평균을 구한다. (책에 있는 code는 동작되지 않음, 오타) 
    def mse(y, y_hat):
    return ((y - y_hat)**2).mean()
  • 평균제곱오차2 
    def mse_val(y, predict_result):
    return mse(np.array(y), np.array(predict_result))
  • 계산값 출력 
    predict_result = [ ]
for i in range(len(x)):
   predict_result.append(predict(x[i]))
   print("공부한 시간 = %.f, 실제 점수 = %.f, 예측 점수 = %.f" %(x[i], y[i], predict(x[i])))
print("mse 최종값:" + str(mse_val(y, predict_result)))
