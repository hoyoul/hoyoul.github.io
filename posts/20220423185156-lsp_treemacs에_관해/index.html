lsp treemacs에 관해

treemacs

  • File을 관리할 수 있다.
  • projectile과 연동해서 project를 관리할 수 있다.
  • workspace 개념을 사용해서 관리한다.

workspace and Projects

  • workspace: the collections of projects의 name만 가지고 있다.
  • project: project는 name과 location을 가지고 있다.
  • workspace를 만들고, workspace에 project를 추가 삭제할 수 있다. 물론 workspace는 switch가 가능하다.

사용법

workspace

개념

  • treemacs는 workspace 개념을 사용해서 window에 보여준다는 점에서 file explorer와 다르다.
  • treemacs를 실행하면 현재 사용하는 buffer가 포함된 project를 찾고, project가 포함된 workspace를 보여준다.

사용법

  • 일반적인 사용법은 다음과 같다. workspace에 여러 project를 추가할 수 있는데, 나는 하나만 쓴다. default workspace를 하나만 사용하고, 여기에 treemacs-add-project-to-workspace를 사용해서 project를 추가한다. 그러면 default workspace로 이동하면 추가된 project별로 directory가 있는 것을 볼 수 있다. 이것을 통해서 이동한다. 물론 이동한 폴더에서 subfolder를 생성하는 것은 c d 명령어로 만들고, c f로 파일을 만들어서 사용한다. treemacs로 우리가 할려는 것은 빠르게 원하는 파일이나 폴더에 접근하는 file explorer 정도의 기능을 원하기 때문이다. workspace를 통해서 원하는 폴더(project)를 모아놨기 때문에 더 접근성이 좋을 뿐이지, 여기서 어떤 굉장한 기능을 요구하는 것이 아니기 때문이다. 가끔 treemacs-edit-workspace를 통해서 project를 추가하거나 수정할 수도 있다.
  • 좀 더 확장해서 사용할려면, work, study,이런식으로 category별 workspace를 이용해서 작업할 수도 있다. 또한 tramp를 사용해서 원격접속하는 경우, 그에 맞는 workspace를 만들수도 있다.여러개의 workspace를 사용한다면, treemacs-tab-bar를 사용하는것도 좋은 방법이다.
  • (treemacs-edit-workspaces): workspace를 수정할 수 있다.
  • (treemacs-switch-workspaces): workspace를 선택할 수 있다.
  • (treemacs-next-workspace): 다음 workspace로 이동
  • (treemacs-create-workspace): workspace 생성(name만 생성된다)
  • (treemacs-add-project-to-workspace): project를 workspace에 추가한다.

  • (M-x treemacs): 왼쪽 창을 만들고, file explorer를 보여준다. toggle이 가능하다. 설정시에 keybinding을 나는 C-x t t로 했다.
  • (M-x treemacs-select-window): 가장 많이 사용되는 기능이다. 왼쪽창에 file explorer로 이동하는 기능이다. C-x o로 이동되지 않고 위명령어로 이동할 수 있다. keybinding을 M-0으로 했다.
  • (c d): treemacs에서 directory 만들기
  • (c f): treemacs에서 file 만들기
  • (?): treemacs window가 보여진 상태에서 cursor를 해당 윈도우에 이동한 후 ?를 입력하면 manipulate할 수 있는 helpful화면이 나온다.
