개괄적 설명

2017년도에 카카오에서 코테로 사람을 뽑음. 왜 알고리즘이 필요한가?에 대해서 설명을 하는데, problem solving skill때문이라고 한다. 코딩문제를 기계적으로 풀기보단 왜? 이렇게 풀어야 하고 어떤 장점이 있는지를 이책에 담을려고 노력했다고 함.

python

cpython을 사용, cpython은 c로 만들어진 python, python을 설치하면 사용되는 interpreter가 cpython이라고 보면 됨. jython이란것도 있음. java로 만든 python.

coding convention tools.

협업을 많이하는 python은 coding convention이 중요하다. 다음과 같은 tool이 있다.

pep(Python Enhanced Properties)8 사용(style checker) - flake8

python으로 program을 작성할때의 규약(coding convention)이라고 보면 됨. warning message로 알려줌.

  • indent: 4 space
  • 함수와 함수 사이: 2 blanks
  • line의 최대길이: 79

pep8은 옛날이름이고 지금은 pycodestyle로 개발 진행되고 있다. pep8이나 pycodestyle은 guideline만을 제시한다. flake8이라는 checker라는 program이 pep8을 수행하는 tool이라고 보면된다.

pylint

pylint는 flake8이라는 tool이 하는 formatting error도 처리하지만, syntatic error도 checking한다.

black formatter

pep8에서 warning message를 보여주지만, 고쳐주진 않는다. 고쳐주는 tool

요약

checker로는 pylint, flake8을 사용하고, reformatter로 black을 사용하면 된다.

emacs에서 사용법

pylint, flake8과 같은 checker는 flycheck이라는 모든 언어의 checking을 담당하는 package에서 처리한다. flycheck은 python이외의 다른언어에서도 사용할 수 있는 범용적인 tool이다. M-x flycheck-verify-setup을 통해서 원하는 checker를 선택할 수 있다. style만 check할려면, pep8만 선택하고, syntax error checking도 하려면 pylint도 설정한다.
